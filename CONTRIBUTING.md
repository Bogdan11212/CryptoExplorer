
# Contributing to CryptoExplorer

Thank you for your interest in contributing to CryptoExplorer! We welcome contributions from the community.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/CryptoExplorer.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `npm install`
5. Start development server: `npm run dev`

## Development Guidelines

### Code Style

- Use TypeScript for all new code
- Follow existing code formatting patterns
- Use meaningful variable and function names
- Add comments for complex logic

### Component Guidelines

- Keep components focused and single-purpose
- Use TypeScript interfaces for props
- Follow the existing component structure in `client/src/components/`
- Ensure components are responsive and accessible

### Commit Messages

Use clear and descriptive commit messages:

```
feat: Add wallet balance chart
fix: Resolve transaction filtering bug
docs: Update API documentation
style: Format code with prettier
refactor: Simplify block data fetching
```

### Testing

- Test your changes across different networks (BTC, ETH, BNB, TRON, TON, LTC)
- Verify responsive design on mobile and desktop
- Check both light and dark themes
- Ensure accessibility standards are met

## Pull Request Process

1. Update the README.md with details of significant changes
2. Ensure your code follows the project's coding standards
3. Test thoroughly before submitting
4. Create a pull request with a clear description of changes
5. Link any related issues

### PR Title Format

```
[Type] Brief description
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

Example: `[feat] Add transaction filtering by date range`

## Code Review

All submissions require review. We aim to review PRs within 48 hours.

## Questions?

Feel free to:
- Open an issue for discussion
- Contact us at [ceo@idevs.fun](mailto:ceo@idevs.fun)

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Focus on the code, not the person
- Help others learn and grow

Thank you for contributing!
